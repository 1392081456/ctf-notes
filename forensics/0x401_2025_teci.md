# 0x401 CTF 2025 — TECI (Trojan + Reverse + Network Forensics)

> **All flags (no `flag{}` wrapping required by ruleset):**
> 1. `8080` (phishing site port)
> 2. `tools_setup.exe` (malware name)
> 3. `943eb6511d63b199a9392b34b3e5e968afd6b782d7eac4857cd392a267679546` (SHA256)
> 4. `TheSecretKeyyyy` (RC4 key)
> 5. `9000` (C2 listener port)
> 6. `hostname` (first attacker command)
> 7. `Win7-2025HGRMOD` (victim hostname)
> 8. `bbba1a754bde7e6aad85f901ead60067` (important.txt MD5)
>
> The challenge that taught me **the discriminator for layered ciphers is not in the binary's metadata — it's in the protocol**. The malware metadata exposed both `Rc4Cipher` and `XorCipher` class names plus two key strings (`TheSecretKeyyyy` and a GUID). The natural assumption is that `Rc4Key = "TheSecretKeyyyy"` (the most "RC4-key-shaped" string). It's backwards: **the GUID is the RC4 key, and the ASCII string is the XOR key**. I lost 20 minutes burning the wrong combination before brute-force enumerating all four key/cipher pairings exposed the right one.

## 3. Q4 + Q5 — RC4 key + listener port

Static string extraction (UTF-16LE around the embedded `Server.dll` metadata at `0x935a00`):

```
@0x935c58: 'Client connected: '
@0x935c7e: '32662c04-a363-4288-8131-f7912004f0f6'   ← GUID
@0x935cc8: 'TheSecretKeyyyy'                         ← Looks like the RC4 key
@0x935cf0: '] Decrypted command: '
@0x935d1c: 'cmd.exe'
@0x935da2: 'Server listening on 0.0.0.0:9000'        ← Q5
```

Combined with the metadata field names (`Rc4Cipher`, `Rc4Key`, `XorCipher`, `XorKey`), the **intuitive but wrong** assignment is:

- RC4 key = `TheSecretKeyyyy`
- XOR key = GUID

The challenge label "RC4 加密算法的密钥" says **the answer for Q4 is `TheSecretKeyyyy`**. That answer is the string the question writer used. But for protocol decryption to actually work (§5), the **runtime mapping is reversed**.

→ Q4: **`TheSecretKeyyyy`**, Q5: **`9000`**

## 4. C2 protocol

Each direction is a stream of length-prefixed encrypted messages:

```
[4 bytes LE length] [length bytes ciphertext]
```

Both directions use the same dual-cipher pair: **RC4 then XOR (encrypt)** / **XOR then RC4-decrypt (decrypt)**.

## 5. The key swap — the trap

Burning hours on the wrong assignment, I finally brute-forced all 4 combinations of `(RC4 key, XOR key)` against the first message `b30ba1210872a861`:

```python
import itertools
for rc4_k, xor_k in itertools.permutations([b'TheSecretKeyyyy', GUID_str]):
    plain = rc4(cipher, rc4_k)
    plain = bytes(a ^ xor_k[i % len(xor_k)] for i, a in enumerate(plain))
    print(rc4_k, '+', xor_k, '→', plain)
```

→ Only **RC4 with GUID** + **XOR with `TheSecretKeyyyy`** produces ASCII:

```
RC4(b'32662c04-...') then XOR(b'TheSecretKeyyyy') → b'hostname'
```

**Lesson**: the field names in metadata aren't authoritative for which string is "the RC4 key" vs "the XOR key". If two ASCII strings of similar length are both candidates, try **all permutations**. It's 2 keys × 2 ciphers = 4 combinations, microseconds.

## 7. Decoder script (complete)

```python
import subprocess, struct

def rc4(data, key):
    S = list(range(256)); j = 0
    for i in range(256):
        j = (j + S[i] + key[i % len(key)]) % 256
        S[i], S[j] = S[j], S[i]
    i = j = 0; out = bytearray()
    for c in data:
        i = (i + 1) % 256
        j = (j + S[i]) % 256
        S[i], S[j] = S[j], S[i]
        out.append(c ^ S[(S[i] + S[j]) % 256])
    return bytes(out)

RC4_K = b'32662c04-a363-4288-8131-f7912004f0f6'
XOR_K = b'TheSecretKeyyyy'

def decrypt(c):
    r = rc4(c, RC4_K)
    return bytes(a ^ XOR_K[i % len(XOR_K)] for i, a in enumerate(r))

def parse(direction_filter):
    out = subprocess.check_output([
        "tshark", "-r", "traffic.pcapng",
        "-Y", f"tcp.stream == 2 and {direction_filter}",
        "-T", "fields", "-e", "tcp.payload"
    ], stderr=subprocess.DEVNULL).decode()
    raw = b''.join(bytes.fromhex(l.strip()) for l in out.splitlines() if l.strip())
    msgs, pos = [], 0
    while pos + 4 <= len(raw):
        L = struct.unpack('<I', raw[pos:pos+4])[0]
        pos += 4
        if L > 0x10000 or pos + L > len(raw): break
        msgs.append(raw[pos:pos+L]); pos += L
    return msgs

print("=== ATTACKER COMMANDS ===")
for i, m in enumerate(parse("ip.src == 192.168.20.1")):
    print(f"#{i}: {decrypt(m).decode('utf-8', errors='replace')}")

print("\n=== VICTIM RESPONSES ===")
for i, m in enumerate(parse("ip.src == 192.168.20.135")):
    if m: print(f"#{i} ({len(m)}B): {decrypt(m).decode('utf-8', errors='replace')[:200]}")
```

## 8. Traps and lessons learned

| Trap | Why it cost me | Resolution |
|---|---|---|
| Assumed `TheSecretKeyyyy` = RC4 key | Field name `Rc4Key` adjacent in metadata + key shape feels right | Brute force all `(rc4_key, xor_key)` permutations against first ciphertext when in doubt — 4 trials only |
| Stopped parsing on `length=0` | First `cd` command returns empty output → 0-length response | `cd` / silent commands legitimately produce `length=0` responses. Don't terminate on zero, only on out-of-range |
| Tried RC4 with `pycryptodome` | No `ARC4` module by default name; `Crypto.Cipher.ARC4` is the path | Or just write RC4 in 15 lines — it's that small |
| 67 MB binary scared me | Assumed PyInstaller / Electron | `.NET 8 NativeAOT` is normal for small C# programs — runtime + framework adds ~60 MB |
| Looked for embedded `Server.dll` PE | NativeAOT doesn't embed managed assemblies | The C# code is compiled to native; only metadata strings (class/method/string-constant names) survive in `.rdata` |

## 9. Methodology takeaways

1. **`.NET NativeAOT` recon**: look for `mscoree.dll`, `hostpolicy`, `CLRJitAttachState`, `Server.dll` in strings. Metadata strings (UTF-16LE) survive in `.rdata` around the `Server.dll` reference and are the goldmine — they include user class names, field names, and string constants.
2. **When metadata exposes 2 key candidates, always permute them against the cipher**. The naming convention isn't authoritative.
3. **Length-prefix protocol parsers must accept length=0**. Silent commands (`cd`, `set`, etc.) return empty output but still send the 4-byte length frame.
4. **Two-layer cipher detection**: if `cipher → RC4(K1)` alone produces high-entropy bytes, try `→ XOR(K2)` next. Order matters for chained ciphers.
5. **`certutil -hashfile` is the go-to Windows hash command**: when an attacker uses it, that's deliberate — the response contains the answer formatted with spaces (`bb ba 1a 75...`) that needs to be stripped/joined.
